# Разработка A/B-тестирования и анализ результатов

## 📌 Описание проекта

Анализ A/B-теста для развлекательного приложения с функцией «бесконечной» ленты.  
Цель — оценить влияние нового алгоритма рекомендаций на вовлечённость пользователей.

---

## 🎯 Задачи проекта

### 1. Работа с историческими данными (EDA)
- Загрузить и изучить исторические данные о сессиях пользователей
- Проанализировать число регистраций по дням
- Проанализировать число просмотренных страниц в первых сессиях
- Рассчитать долю пользователей, просмотревших 4+ страниц в первой сессии

### 2. Подготовка к тесту
- Рассчитать необходимый размер выборки
- Рассчитать длительность A/B-теста на основе дневного трафика

### 3. Мониторинг A/B-теста
- Проверить распределение пользователей по группам
- Проверить независимость групп (отсутствие пересечений)
- Проверить равномерность распределения по устройствам и регионам

### 4. Проверка результатов A/B-теста
- Сформулировать нулевую и альтернативную гипотезы
- Определить целевую, прокси- и барьерные метрики
- Рассчитать долю успешных первых сессий в группах A и B
- Проверить статистическую значимость (Z-тест пропорций)
- Сформулировать выводы и рекомендации

---

## 📊 Результаты

- Размер выборки на группу: 41 040 пользователей
- Длительность теста: 9 дней
- Группы сбалансированы, пересечений нет, распределение по устройствам и регионам равномерное
- Доля успешных сессий в группе A: 31.57%
- Доля успешных сессий в группе B: 31.47%
- Разница (B - A): -0.11%
- p-value = 0.578 > 0.05 → нулевая гипотеза подтверждается

**Вывод:** Новый алгоритм рекомендаций не оказал статистически значимого влияния на вовлечённость пользователей. Внедрение не рекомендуется.
