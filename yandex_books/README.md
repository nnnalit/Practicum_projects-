# Проект «Яндекс Книги»

## 📌 Описание проекта

Анализ данных сервиса Яндекс Книги: расчёт метрик в SQL, проверка гипотезы о времени активности пользователей из Москвы и Санкт-Петербурга, анализ результатов A/B-тестирования для интернет-магазина BitMotion Kit.

---

## 🎯 Задачи проекта

### Часть 1. Проверка гипотезы (SQL + Python)

**Задача:**  
Проверить гипотезу о том, что пользователи из Санкт-Петербурга проводят в среднем больше времени за чтением и прослушиванием книг в приложении, чем пользователи из Москвы.

**Этапы:**
1. Подготовка данных в SQL:
   - Соединение таблиц `audition`, `content`, `author`, `geo`
   - Агрегация суммарного времени активности по пользователям
   - Фильтрация пользователей из Москвы и Санкт-Петербурга
2. Проверка гипотезы в Python:
   - Загрузка данных из файла `yandex_knigi_data.csv`
   - Проверка дубликатов в идентификаторах пользователей
   - Сравнение размеров групп и статистик
   - Проведение одностороннего t-теста для двух выборок
   - Интерпретация p-value и формулировка вывода
3. Составление аналитической записки:
   - Выбранный тип t-теста и уровень статистической значимости
   - Результат теста (p-value)
   - Вывод на основе p-value
   - Возможные причины полученных результатов

### Часть 2. Анализ результатов A/B-тестирования (BitMotion Kit)

**Задача:**  
Оценить корректность проведения A/B-теста и проанализировать его результаты.

**Гипотеза:**  
Упрощение интерфейса приведёт к тому, что в течение семи дней после регистрации конверсия зарегистрированных пользователей в покупателей увеличится как минимум на три процентных пункта.

**Параметры теста:**
- Название теста: `interface_eu_test`
- Группы: A (контрольная), B (новый интерфейс)

**Этапы:**
- Загрузка данных участников теста и событий
- Проверка корректности проведения теста
- Анализ результатов эксперимента
- Формулировка выводов и рекомендаций для бизнеса
