# Финальный проект «Яндекс Афиша»

## 📌 Описание проекта

Полный цикл анализа данных сервиса Яндекс Афиша за период с 1 июня по 31 октября 2024 года.  
Проект состоит из двух частей: разработка дашборда в DataLens и исследовательский анализ данных в Python с проверкой гипотез.

---

## 🎯 Задачи проекта

### Часть 1. SQL + DataLens (дашборд)

**Цель:** Разработать аналитический дашборд для отслеживания ключевых бизнес-показателей.

**На дашборде отображены:**

**Группа индикаторов (за весь период):**
- Итоговая выручка
- Общее количество заказов
- Средняя выручка с одного заказа
- Число уникальных покупателей

**Линейные графики (недельная динамика):**
- Выручка с заказа
- Количество заказов
- Средняя выручка с заказа

**Структура выручки (за весь период):**
- В разрезе типа устройств
- В разрезе типа мероприятия

**Таблицы топ-сегментов:**
- Популярные регионы
- Популярные события
- Популярные площадки

**Для каждого сегмента рассчитаны:**
- Общая выручка
- Количество заказов
- Среднее количество билетов в заказе
- Средняя выручка с одного билета

**Фильтры на дашборде:**
- По валюте (по умолчанию — рубли)
- По дате (по умолчанию — 01.06–31.10.2024)
- По типу события
- По региону

**Результат:** Создан дашборд, добавлена аналитическая записка на отдельной вкладке.

---

### Часть 2. Python (EDA и проверка гипотез)

**Цель:** Провести исследовательский анализ данных и проверить гипотезы о поведении пользователей.

**Шаг 1. Загрузка и знакомство с данными**
- Загрузка датасетов `final_tickets_orders_df.csv`, `final_tickets_events_df.csv`, `final_tickets_tenge_df.csv`
- Оценка объёма, корректности и соответствия описанию

**Шаг 2. Предобработка данных**
- Проверка пропусков (кроме `days_since_prev`)
- Анализ категориальных и количественных значений
- Обработка выбросов по 99-му процентилю
- Проверка явных и неявных дубликатов
- Преобразование типов данных
- Создание новых столбцов:
  - `revenue_rub` — выручка в рублях (с учётом курса тенге)
  - `one_ticket_revenue_rub` — выручка с одного билета
  - `month` — месяц заказа
  - `season` — сезон (лето/осень/зима/весна)
  - `weekday` — день недели
  - `date` — дата заказа
  - `hour` — час заказа

**Шаг 3. Исследовательский анализ данных**

**3.1. Анализ распределения заказов по сегментам и сезонность**
- Динамика количества заказов по месяцам (рост 190% от июня к октябрю)
- Сравнение распределения заказов летом и осенью по типам мероприятий, устройствам, возрастным рейтингам
- Анализ изменения средней выручки с одного билета по типам мероприятий (стендап +5.8%, театр -17.8%)

**3.2. Осенняя активность пользователей**
- Динамика по дням: число заказов, DAU, среднее число заказов на пользователя, средняя стоимость билета
- Анализ недельной цикличности (пик — вторник, минимум — воскресенье)

**3.3. Популярные события и партнёры**
- Регионы: количество мероприятий и заказов (лидер — Каменевский регион, 31% заказов)
- Билетные партнёры: число мероприятий, заказов, выручка (лидер — «Билеты без проблем», 22% заказов)

**Шаг 4. Статистический анализ данных (проверка гипотез)**

**Гипотеза 1 (проверена, подтверждена):**
- Среднее количество заказов на одного пользователя мобильного приложения выше, чем у пользователей стационарных устройств
- Mann-Whitney U-test, p-value ≈ 0.000

**Гипотеза 2 (проверена, подтверждена):**
- Среднее время между заказами пользователей мобильных приложений выше, чем у пользователей стационарных устройств
- Mann-Whitney U-test, p-value ≈ 0.000

**Шаг 5. Общий вывод и рекомендации**
- Обобщение результатов анализа
- Рекомендации для бизнеса

## 📊 Результаты

### Ключевые цифры
- **Период анализа:** июнь – октябрь 2024
- **Заказов:** 287 782 (после предобработки)
- **Общая выручка:** 149 271 204 руб.
- **Средний чек:** 518.70 руб.
- **Уникальных пользователей:** 21 422

### Основные выводы
- Рост заказов осенью: **+190%** (июнь → октябрь)
- Основной канал: **мобильные устройства** (~80% заказов)
- Пик активности: **вторник** (неожиданно, не выходные)
- Выручка на билет снижается осенью (театр -17.8%, концерты -12%)
- Мобильные пользователи совершают больше заказов, но с большими интервалами

### Статистические гипотезы
| Гипотеза | Результат |
|----------|-----------|
| Mobile users делают больше заказов | ✅ подтверждена (p < 0.001) |
| Mobile users покупают реже | ✅ подтверждена (p < 0.001) |
